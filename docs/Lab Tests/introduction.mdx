# Introduction

Vital provides a simple way to order and manage lab tests. We handle all the logistics, packing and processing between a user taking the test and the results being available through our API.

## 1. Types of tests

As a first step, we need to define what type of testing you want to offer to your users. We will have an introduction call to go over the test, projected volumes, customizations for kits and other questions you might have. Once we have all this information, we can begin setting up the test kits and distribution. Finally, we can set up your integration and dashboard to start ordering tests via our API.

Once this is done, you will have the tests available via our API:

```bash
curl --request GET \
     --url {{BASE_URL}}/v2/testkit/ \
     --header 'Accept: application/json' \
     --header 'x-vital-api-key: <API_KEY>' \
     --header 'Content-Type: application/json' \
```

<br />
<br />

This will return a list of all the available tests to you. They include a description, turnaround time and price:

```json
{
  "testkits": [
    {
      "id": "7cd5e01a-9sa2-4bcs-a6f3-4cccacb74824",
      "name": "Brand Respiratory Allergen",
      "description": "Respiratory Allergens",
      "markers": [],
      "turnaround_time_lower": 4,
      "turnaround_time_upper": 14,
      "price": 216.0
    }
  ]
}
```

<br />
<br />

You will then use the test's `id` when creating an order.

## 2. Ordering Tests

Ordering a test for your user, is as simple as making an API call:

```bash
curl --request POST \
     --url {{BASE_URL}}/v2/order/ \
     --header 'Accept: application/json' \
     --header 'x-vital-api-key: <API_KEY>' \
     --header 'Content-Type: application/json' \
     --data '
{
     "testkit_id": "7cd5e01a-9sa2-4bcs-a6f3-4cccacb74824",
     "patient_address": ...,
     "patient_details": ...,
     "user_id": ...
}
'
```

<br />
<br />

Once an order status change, we will send you a webhook:

```json
{
  "id": "ADdee02a-9sa2-4bcs-a6f3-4cccacb74831",
  "team_id": "",
  "created_on": "2020-01-01T00:00:00.000Z",
  "updated_on": "2020-01-50T00:00:00.000Z",
  "status": "with_customer",
  "testkit_id": "7cd5e01a-9sa2-4bcs-a6f3-4cccacb74824",
  "testkit": {
    "id": "7cd5e01a-9sa2-4bcs-a6f3-4cccacb74824",
    "name": "Brand Respiratory Allergen",
    "description": "Respiratory Allergens",
    "markers": [],
    "turnaround_time_lower": 4,
    "turnaround_time_upper": 14,
    "price": 216.0
  }
}
```

<br />
<br />

The most important field is the `status`. It tells in what stage the order is. These are the currently possible values:

```text
"ordered"
"transit_customer"
"out_for_delivery"
"with_customer"
"transit_lab"
"delivered_to_lab"
"processing_lab"
"completed"
"failure_to_deliver_to_customer"
"failure_to_deliver_to_lab"
"cancelled"
"do_not_process"
"unknown"
```

<br />
<br />

You can also fetch an order by its `id`:

```bash
curl --request GET \
     --url {{BASE_URL}}/v2/order/<order_id> \
     --header 'Accept: application/json' \
     --header 'x-vital-api-key: <API_KEY>' \
     --header 'Content-Type: application/json' \
```

<br />
<br />

Or all the orders you have made for a period of time:

```bash
curl --request GET \
     --url {{BASE_URL}}/v2/order?start_date={{START_DATE}}&end_date={{END_DATE}} \
     --header 'Accept: application/json' \
     --header 'x-vital-api-key: <API_KEY>' \
     --header 'Content-Type: application/json' \
```

<br />
<br />

Finally, you can cancel an order:

```bash
curl --request POST \
     --url {{BASE_URL}}/v2/order/<order_id>/cancel \
     --header 'Accept: application/json' \
     --header 'x-vital-api-key: <API_KEY>' \
     --header 'Content-Type: application/json' \
```
